# 4. 디버깅 보고서 (10점)

> 💡 **디버깅이란?** 프로그램에서 발생한 오류(버그)를 찾아내고 고치는 과정입니다.

---

## 📝 작성 정보
- **작성 날짜:** 2025년 _11_월 _27_일
- **작성자:** 김용찬

---

## 🐛 발견한 오류 #1
없음
### 1) 어떤 문제가 발생했나요?
(예: 프로그램을 실행하면 이상한 값이 나왔어요 / 컴파일이 안 돼요)

```
문제 상황을 간단히 설명하세요.
```

### 2) 오류가 발생한 코드
```c
// 문제가 있던 코드를 여기에 복사하세요
```

### 3) 왜 오류가 생겼나요?
(예: 세미콜론(;)을 빼먹었어요 / 변수 이름을 잘못 썼어요)

```
원인을 간단히 적어보세요.
```

### 4) 어떻게 고쳤나요?
```c
// 고친 코드를 여기에 적어주세요
```

### 5) 고친 후 결과
- [ ] ✅ 잘 작동해요!
- [ ] ❌ 아직 문제가 있어요

## 🧪 프로그램 테스트 결과

### 테스트 1
- **무엇을 테스트했나요?** (예: 학생 정보 입력)
- **어떻게 테스트했나요?** (예: 이름에 "홍길동", 나이에 20을 입력했어요)
- **결과:** ✅ 성공 / ❌ 실패

### 테스트 2
- **무엇을 테스트했나요?** 
- **어떻게 테스트했나요?** 
- **결과:** ✅ 성공 / ❌ 실패

## 💭 디버깅하면서 배운 점

이번 프로젝트에서 오류를 고치면서 새롭게 알게 된 점이나 느낀 점을 자유롭게 적어보세요.

```
(예시)
- 세미콜론(;)을 빼먹으면 컴파일 오류가 난다는 것을 배웠어요.
- scanf를 쓸 때 &를 꼭 써야 한다는 걸 알았어요.
- 에러 메시지를 잘 읽으면 문제를 더 쉽게 찾을 수 있었어요.
```

프로그램이 잘 작동하는지 최종 확인해보세요!

- [ ] 컴파일이 잘 되나요? (오류 메시지가 없나요?)
- [ ] 프로그램이 실행되나요?
- [ ] 메뉴가 제대로 나오나요?
- [ ] 입력한 값이 정상적으로 처리되나요?
- [ ] 모든 기능이 잘 작동하나요?
